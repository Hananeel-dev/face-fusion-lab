Technical Notes

How the composite and similarity score are produced.

This project uses face landmarks as the common reference frame. The score is a visual-compositing similarity estimate, not biometric identity verification.

Back to App API Docs

1. Face detection

Each uploaded image is decoded with OpenCV. MediaPipe Face Landmarker estimates one face and its landmarks. If landmarks are not found immediately, the app falls back to OpenCV Haar face detection for the crop and then tries landmarks again.

2. Crop and alignment

The detected face bounding region is expanded into a square crop. The crop is rotated so the eye landmarks are level, then resized to a shared 256 by 256 pixel canvas.

3. Shared average shape

The two aligned landmark sets are averaged point by point. This mean landmark set becomes the target geometry for the composite.

4. Triangle warp

The app builds a Delaunay triangle mesh over the target landmarks and image boundary points. Each triangle from both faces is affine-warped into the target shape, which keeps eyes, nose, mouth, jaw, and face outline in comparable positions.

5. Composite blend

The two warped face images are blended with equal weight. The result is written as a runtime image under static storage so the browser can display it.

6. Similarity metrics

Landmark geometry compares normalized landmark structure. Warp similarity estimates how little each face needed to deform to reach the shared shape. Texture similarity compares grayscale detail after alignment. Composite agreement measures remaining pixel-level variation after warping.

7. Limitations

Scores are affected by lighting, camera angle, expression, facial hair, glasses, blur, and crop quality. A high score means the images are visually easy to align and blend; it does not prove the two images show the same person.
